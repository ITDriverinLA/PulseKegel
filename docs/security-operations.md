# Security operations

## Required Replit Secrets

The analytics administration routes fail closed unless both of these secrets
are configured:

- `ANALYTICS_ADMIN_USERNAME`
- `ANALYTICS_ADMIN_PASSWORD`

Use a unique password generated by a password manager. Do not commit either
value to GitHub. After deployment, visiting `/analytics` opens the browser's
username-and-password prompt.

`INVALIDATE_CACHE_TOKEN` is also required for the sitemap cache-invalidation
endpoint. If it is absent, that endpoint returns `503` instead of accepting an
unauthenticated request.

## Analytics reset

The reset remains available to an authenticated administrator:

```sh
curl --user "ADMIN_USERNAME:ADMIN_PASSWORD" \
  --request DELETE https://pulsekegel.com/api/analytics/reset
```

The reset permanently deletes all analytics events. Confirm the target domain
and credentials before running it.

## Public endpoint limits

- Analytics ingestion accepts only the event types and fields used by the app,
  with a maximum of 20 events per request and 60 requests per IP per minute.
- Weekly AI reviews accept bounded, validated workout data. They are limited to
  10 requests per IP per hour and 250 requests globally per hour.
